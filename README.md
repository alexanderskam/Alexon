<h1>Мессенджер Alexon</h1>
<h2>Что представляет из себя проект?</h2>
<p>Проект представляет из себя Fullstack мессенджер. Приложение было написано с целью изучения работы с WebSocket, выработки паттернов для применения их на коммерческих проектах, практики работы на современном стеке, влючающем все необхлдимые технологии</p>
<p><strong>Деплой: </strong>https://alexon-sigma.vercel.app</p>
<h2>Технологический стек</h2>
<p>Backend: Node.js, TypeScript, Express для разработки HTTP-части сервера, Socket.io для разработки WS-части сервера, MongoDB для хранения данных, mongoose для взаимодействия базы с сервером. </p>
<p>Frontend: React, TypeScript как основа, React Router для навигации, Axios для http запросов, Socket.io для realtime, Redux Toolkit использовался для состояний на клиенте, Tanstack Query использовался в качестве инструмента для работы с запросами к серверу и кэширования</p>
<h2>Функционал приложения</h2>
<h3>Клиент:</h3>
<p><strong> Пользователю доступны страницы регистрации, логина, поиска пользователей, страница с чатами пользователя, страница с информацией об аккаунте, кнопка логаута. Подробнее о каждой из страниц:</strong></p>
<p><strong>Регистрация:</strong> Проводится валидация каждого из полей на бэкенде, если что-то не так, приходит ошибка, которую отправляет middleware, далее она выводится на экран с помощью библиотеки react-toastify. Благодаря
данному функционалу, пользователь получает информативные уведомления о проблемах с регистрацией, как по его вине, так и по вине сервера. При submit формы кнопка отключается, предотвращая повторные запросы и ошибки. При успешной
регистрации пользователю на почту приходит письмо активации. </p>
<p><strong>Логин:</strong> Включает в себя все UX-решения регистрации</p>
<img width="1909" height="961" alt="login" src="https://github.com/user-attachments/assets/f7d41e58-081b-4544-9909-b3cbaf37cb7f" />
<p><strong>Поиск пользователей:</strong> Разработан интерфейс с двумя полями ввода, одно для Username, второе для почты, они работают одновременно, можно вводить в оба поля и результат будет включать в себя фильтры 
из обоих полей. Для лучшего UX пользователи загружаются по принципу пагинации, внизу есть кнопка, догружающая пользователей. Каждый введенный символ отправляет на сервер запрос, который выполнится только если пользователь больше 
ничего не введет, в противном случае на сервер пробрасывается сигнал, который в случае изменения инпутов отменяет предыдущий запрос. К каждому найденному пользователю прикреплена интерактивная иконка, при нажатии
на которую пользователь заводит чат с найденным и происходит редирект в чат, а если возникла ошибка, пользователь получит информативное уведомление о ней. При обработке действия начала чата остальные такие конпки отключаются, а пользователь получает на экране предупреждение о загрузке.</p>
<img width="1916" height="959" alt="users" src="https://github.com/user-attachments/assets/13a2671d-5246-46fc-84f8-32e88d27a9b4" />

<p><strong>Чаты:</strong> Страница, вдохновленная принципами telegram, повторяющая дизайн последнего. Пользователь может свободно переключать чаты, если он попал на страницу путем создания чата, он сразу попадет в чат с пользователем,
если он зашел на нее сам(через хэдер), пользователю будет предложено выбрать чат. Так же есть кнопка начала нового чата, которая редиректит пользователя на страницу поиска. К каждому чату прилагаются индикаторы онлайна собеседников
и непрочитанных сообщений от них, индикаторы реализованы с помощью WebSocket. Для каждого чата есть контекстное меню, в котором можно удалить выбранный чат, при этом чат удалится у обоих пользователей в реальном времени, реализовано
опять же с помощью ws. Касаемо самого чата, сообщения представляют из себя объекты со статусом прочитанного, с датой отправления и возможностью удаления. В чате есть индикатор печати, если собеседник вам что-то пишет.
Страница оптимизирована, все ошибки которые возникают появляются в виде уведомления, все состояния загрузки отображаются.</p>
<img width="1912" height="965" alt="chats" src="https://github.com/user-attachments/assets/65110eac-08a0-47e5-b1ab-ee79f8fdcc6e" />
<p><strong>Информация об аккаунте:</strong> Страница с выводом информации о пользователе, с кнопкой удаления аккаунта, которая так же удаляет все чаты в реальном времени у всех собеседников, а так же инвалидирует кэш на странице поиска у всех
онлайн пользователей, а после редиректит пользователя на регистрацию и чистит состояния. Если аккаунт не активирован через почту, будет предложено активировать его, на почту отправится рассылка с ссылкой активации. 
Все действия отправляются пользователю в виде уведомления, все кнопки отключаются на время взаимодействия с сервером.</p>
<img width="1916" height="950" alt="profile" src="https://github.com/user-attachments/assets/0627a8c0-3206-4e82-aa0f-5cf42c968dfb" />
<strong>В приложении развит UX, все мутации происходят по принципу optimistic update, где это возможно и логично, и pessimistic update, случай рассинхрона исключен. </strong>
<p>Клиентская часть строго структурирована, логика и интерфейс сепарированы, для запросов написана api, socket-логика вынесена в отдельные хуки </p>
<p>Все маршруты защищены Protect роутом, проверяющим авторизацию пользователя</p>
<h3>Сервер:</h3>
<p><strong>Структура:</strong> весь бэкенд разделен и сепарирован, отдельный файл запуска, файл роутера, файлы контроллеров, сервисы(user-service, mail-service, token-service и т.д.), модели. Бэкенд реализован по принципам REST. </p>
<p><strong>Авторизация:</strong> Выполнена по принципам jwt-авторизации, без дыр, production уровень, refresh token записывается в http-only куку, access отправляется на клиент в body и хранится в localStorage. Все запросы
защищены middleware, проверяющим валидность access токена, если он не валиден, на клиент приходит ошибка и api пытается сделать refresh, в случае неудачи пользователя редиректит на логин. Socket соединение также защищено middleware. </p>
<h2>Инструкция по запуску: </h2>
<p>git clone https://github.com/alexanderskam/Alexon</p>
<p>cd backend</p>
<p>npm install</p>
<p>npm run dev</p>
<p>cd client</p>
<p>npm install</p>
<p>npm run dev</p>
